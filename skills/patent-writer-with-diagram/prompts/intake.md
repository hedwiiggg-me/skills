# 边界与输入录入（Step 1）

## 用途

在开始撰写交底书前，用**少量问题**收敛案件边界。信息不全时可跳过，由 Agent 后续推断并**注明假设**。

## 可选开场

```
为便于整理技术交底书，先确认几项基本信息；可跳过，将按已有材料推断补齐。
```

## 必填信息

### 基本信息

- **专利名称**：简洁描述方案，如"一种XX功能的实现方法"
- **专利类型**：发明 or 外观
- **发明人姓名**（可多人）
- **联系人 + 联系电话 + email**

### 核心内容（引导用户描述）

- 要解决什么问题？现有方案是什么？现有方案的缺点？
- 本方案的核心创新点是什么？
- 方案的具体实现方式（交互流程、技术实现、视觉呈现）
- 采用本方案的优势

## 汇总

将已确认项用 3–6 行 bullet 复述给用户，再进入 `disclosure_builder.md` 对应的撰写阶段。
