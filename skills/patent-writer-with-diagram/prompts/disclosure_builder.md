# 技术交底书撰写（Step 2）

详细章节范例与模版细节见同目录 **`template_reference.md`**。

## 写作风格

参考 `references/examples.md` 中的真实案例，理解写作风格和深度。写作时优先保留用户自己的产品、场景和品牌信息，不要机械沿用案例中的品牌表述。

## 章节结构

**⚠️ 强制约束：正文必须严格按照以下模板结构撰写，不得省略任何子标题。**

### 第1节 背景（对应文档中的"1 背景"标题）

必须包含以下三个子部分，按顺序撰写：
- **问题的描述**：从用户视角描述使用场景和需求背景，说明用户目前如何解决这个问题（现有方案是什么）。**不涉及任何技术实现，只描述用户行为和体验现状。**
- **现存问题的缺点**：从用户体验角度列出现有方案的缺点（操作繁琐、反馈不及时、认知负担重、信息噪点多等），用编号列表。**不使用技术术语，站在用户感受层面描述。**

### 第2节 发明要点（对应文档中的"2，发明要点"标题）

必须包含以下两个子部分，按顺序撰写：
- **针对上述问题和现有解决方案的缺陷，本方案提出**：一句话概括核心方案，**用产品功能语言描述，不涉及底层技术实现**
- **采用本方案的优势**：从用户体验和使用效率角度列出2-3条具体优势，用编号列表。**聚焦用户获得的价值，不描述技术手段。**

### 第3节 描述（对应文档中的"3，描述"标题）

必须包含以下子部分，按顺序撰写：
- **交互行为描述**：按场景分条，每个场景描述完后**紧跟**该场景对应的截图（图文交织，不允许所有文字写完再集中放图）
  - 每张图必须有 caption 和 annotations（步骤说明）
  - **无论发明专利还是外观专利，第3节均只描述用户视角的交互行为、视觉呈现和操作流程，不涉及任何技术实现细节（如算法、模型、数据结构等）。技术实现内容统一放在第4节。**

### 关于第4节技术内容

- 发明专利：第4节写技术实现细节（算法、流程、数据处理等）
- 外观设计专利：第4节写设计思路与视觉风格

## 文档头部模板

```markdown
# 技术交底书

**案件名称**：[待填写]一种XXX方法及系统

**技术联系人**：
- 姓名：[待填写]
- 电话：[待填写]
- 邮箱：[待填写]

**专利类型**：发明 / 外观设计

---

## 注意事项

（1）交底书应使代理人能看懂，尤其是背景技术和详细技术方案，一定要写得全面、清楚、完整；
（2）技术的公开程度，应以本领域普通技术人员不需付出创造性劳动即可进行实施为准。
（3）在与代理人沟通时，对于代理人咨询的技术问题，应给予回答并认真讲解，并且按要求及时正确地补充相应技术材料。
```

## 脱敏要求

| 类型 | 脱敏方式 | 示例 |
|------|----------|------|
| 业务/行业 | 抽象为通用描述 | 「XX检测」→「多标签分类场景」 |
| 具体分类 | 用 ABC 等替代 | 「类别1、类别2」→「分类A、分类B、分类C」 |
| 具体数值 | 用范围或示例说明 | 「每日 N 人」→「每日一定规模」 |
| 公司/产品 | 不出现具体名称 | 删除或替换为「某系统」 |
