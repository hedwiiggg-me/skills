# 交底书模板参考

## 第1节 背景

### 问题的描述

从用户视角描述使用场景和需求背景，说明用户目前如何解决（现有方案是什么）。

**写作要点**：
- 只描述用户行为和体验现状
- 不涉及技术实现
- 让读者理解"为什么要解决这个问题"

### 现存问题的缺点

从用户体验角度列出缺点，用编号列表：
1. 操作繁琐：需要多个步骤完成
2. 反馈不及时：用户等待时间长
3. 认知负担重：信息展示不直观

## 第2节 发明要点

### 针对上述问题，本方案提出

一句话概括核心方案，用产品功能语言描述。

### 采用本方案的优势

列出2-3条具体优势：
1. **效率提升**：减少操作步骤
2. **体验优化**：更直观的反馈
3. **智能化**：自动处理复杂逻辑

## 第3节 描述

### 交互行为描述

按场景分条，每个场景后紧跟截图。场景格式：
- **场景1：XX功能**
  - 状态描述1：详细说明
  - 状态描述2：详细说明
  - ...
- **场景2：XX功能**
  - 状态描述1：详细说明
  - ...

## 第4节 技术实现（发明专利）

- 第4节技术实现的内容由 Agent 根据交互描述推断补充
- 如果用户提供了技术文档，按文档编写
- 包括：系统架构、处理流程、关键算法、数据结构等

## 命名规范

凡落盘交付的 `.md` / `.docx`，主文件名须为：**`{案件名称}_{YYYYMMDDHHmmss}.md`** 与同名 **`.docx`**。
